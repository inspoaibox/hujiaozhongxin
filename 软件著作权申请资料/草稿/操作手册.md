# 千牛云呼叫中心系统操作手册

## 一、软件概述

千牛云呼叫中心系统 V1.0 面向企业客服、售后服务、投诉处理、电话回访和内部服务台等场景。软件把座席接打电话、客户资料、工单记录、队列等待、主管监控和服务复盘放在同一套系统中，便于企业把电话服务过程记录下来，并让主管及时看到团队接线情况。

软件主要由座席工作台、管理后台和系统服务组成。座席工作台用于客服人员接听来电、发起外呼、查看来电信息、执行保持、转接、三方通话和挂断等操作；管理后台用于主管或管理员查看在线座席、通话中人数、空闲座席、队列等待、今日呼叫统计和呼叫趋势；系统服务负责登录认证、呼叫分配、客户资料、工单、录音、质检、报表、通知和运行维护等工作。

本手册按审核人员能够理解的方式说明软件用途、页面入口、主要操作和结果反馈。文中涉及截图的位置均保留了可见的截图预留文字，后续可以根据实际运行页面替换为软件截图。

## 二、适用对象与运行环境

本软件适用于企业客服座席、座席主管、系统管理员、质检人员和运维人员。座席主要使用座席工作台处理客户电话；主管和管理员主要使用管理后台查看团队运行情况、处理预警和管理基础资料；质检人员根据通话记录、录音和满意度情况检查服务质量；运维人员根据部署和监控资料维护系统运行。

推荐运行环境如下：

| 项目 | 建议配置 |
| --- | --- |
| 客户端设备 | 普通办公电脑或坐席电脑 |
| 浏览器 | Chrome、Edge 或其他现代浏览器 |
| 开发操作系统 | Windows 11 专业版 64位 |
| 运行操作系统 | Windows 10/11、Linux 服务器或支持现代浏览器的客户端环境 |
| 支撑软件 | Node.js、Docker、Docker Compose、Java、浏览器、PostgreSQL、Redis、Kafka、Nginx、FreeSWITCH 等 |
| 网络要求 | 能访问系统服务地址，座席通话时保持网络稳定 |

## 三、软件进入方式

用户通过浏览器访问系统地址。座席进入座席工作台登录页，管理员或主管进入管理后台登录页。输入账号和密码后，系统会验证用户身份，验证通过后进入对应工作页面；验证失败时，页面会提示用户检查账号、密码或联系管理员。

座席登录后通常进入“呼叫处理”页面，页面顶部显示当前座席姓名和状态，中间显示软电话，右侧显示当前提醒和空闲座席。管理员或主管登录后进入实时监控页面，页面显示团队实时指标和图表信息。

## 四、核心功能操作

### 1. 登录认证与角色进入

登录认证用于确认当前使用者身份，并把用户带入对应工作页面。座席、主管和管理员使用各自账号登录后，看到的入口和可办理事项不同。座席主要进入呼叫处理页面，主管和管理员主要进入实时监控与管理页面。

操作步骤：

1. 打开系统访问地址，进入登录页面。
2. 在登录框中输入用户名和密码。
3. 点击登录按钮，等待页面返回验证结果。
4. 登录成功后进入座席工作台或管理后台；登录失败时按照页面提示重新填写。

结果反馈：登录成功后，页面会保存当前登录状态，并显示当前用户可使用的功能入口。退出登录后，系统清除登录状态并返回登录页。

【截图预留：请在此处插入“登录认证与角色进入”页面或操作结果截图。】

### 2. 座席工作台

座席工作台是客服人员日常接打电话的主页面。页面显示“座席工作台”和“呼叫处理”标题，右上角显示当前座席姓名和状态，座席可在这里查看自己是否空闲、是否正在通话，以及当前是否有待处理提醒。

操作步骤：

1. 座席登录后进入呼叫处理页面。
2. 查看顶部状态标签，确认当前状态是否为空闲。
3. 查看软电话区域，准备接听来电或主动拨号。
4. 查看右侧空闲座席列表，转接通话时可选择合适的目标座席。

结果反馈：工作台会集中展示座席当前状态、呼叫处理区域、提醒信息和空闲座席列表。座席完成退出操作后，页面返回登录入口。

【截图预留：请在此处插入“座席工作台”页面截图。】

### 3. 软电话与通话控制

软电话用于完成座席通话操作。座席可以在拨号盘输入号码发起呼出，也可以在来电弹窗中接听客户来电。通话建立后，页面显示客户名称或号码、通话状态和通话时长，并提供保持、静音、转接、三方通话和挂断等按钮。

操作步骤：

1. 发起呼出时，在软电话输入框中填写客户电话号码。
2. 点击拨打按钮，等待页面进入呼叫中或通话中状态。
3. 通话过程中，根据需要点击保持、恢复、静音、转接或三方通话按钮。
4. 通话结束时点击挂断按钮，结束当前通话。

结果反馈：拨号成功后页面显示当前通话信息；保持或恢复后状态标签发生变化；转接成功后当前通话从原座席页面移出；挂断后通话计时停止，座席进入后续整理或空闲状态。

【截图预留：请在此处插入“软电话与通话控制”页面或操作结果截图。】

### 4. 来电弹屏与实时提醒

客户来电时，座席工作台会弹出来电窗口。窗口显示来电号码、客户名称和 VIP 标识等信息，座席可以选择接听或拒接。系统也会向座席推送工单分配、整理超时等提醒，避免重要事项被遗漏。

操作步骤：

1. 座席保持登录状态，并确认当前可以接线。
2. 客户来电后查看弹出的来电窗口。
3. 根据业务情况点击接听或拒接。
4. 接听后查看客户信息，开始处理本次服务；拒接后该呼叫按规则返回等待或重新分配。

结果反馈：接听后页面进入通话中状态，并显示通话计时和通话控制按钮；有新工单或整理超时提醒时，页面会显示通知信息。

【截图预留：请在此处插入“来电弹屏与实时提醒”页面或操作结果截图。】

### 5. 呼叫分配与队列处理

呼叫分配用于把客户来电交给合适的座席处理。系统会根据技能组、座席状态、客户等级和空闲时间等条件选择座席。所有座席忙碌时，客户来电进入等待队列；当座席恢复空闲后，等待队列中的呼叫继续分配。

操作步骤：

1. 客户拨打企业服务热线，来电进入系统。
2. 系统识别来电所属技能组和客户等级。
3. 若有空闲座席，系统把呼叫分配给匹配座席，并在座席页面显示来电弹窗。
4. 若暂无空闲座席，呼叫进入队列，等待后续分配。

结果反馈：呼叫被分配时，座席页面出现来电提醒；排队等待时，主管可在监控页面看到队列等待数量；座席恢复空闲后，系统继续处理队列中的呼叫。

### 6. 座席状态与技能组管理

座席状态用于反映客服人员当前是否可以接线。系统支持空闲、通话中、整理、休息、离线等状态。技能组用于把座席按业务能力分组，例如通用客服、技术支持、投诉处理和 VIP 专席。主管可根据状态和技能组了解团队接线能力。

操作步骤：

1. 主管或管理员进入座席管理相关页面。
2. 查看座席工号、姓名、技能组和当前状态。
3. 根据实际情况调整座席状态、培训模式或技能组归属。
4. 保存后返回列表，检查页面展示是否更新。

结果反馈：座席状态更新后，工作台和监控页面会显示新的状态。进入休息或离线状态的座席不再接收新的客户来电；恢复空闲后可继续参与分配。

### 7. 客户资料与黑名单管理

客户资料用于保存客户姓名、电话、邮箱、地址、VIP 等级和备注信息。座席接听来电时，可根据电话号码查看客户资料。黑名单用于标记不允许接入或需要特殊处理的号码。

操作步骤：

1. 进入客户管理页面，按姓名、电话或 VIP 等级查询客户。
2. 选择客户记录，查看基本资料、历史通话和相关工单。
3. 新客户来电时，填写客户姓名、电话、邮箱、地址和备注后保存。
4. 对需要限制的号码执行黑名单新增、移除或查询操作。

结果反馈：客户资料保存后，后续来电弹屏可显示客户名称和 VIP 标识；黑名单状态变更后，系统在后续来电或查询时按新状态展示。

### 8. 工单创建、分配与状态流转

工单用于记录客户问题、投诉、建议或故障处理事项。座席可在通话中或通话结束后创建工单，填写标题、描述、优先级、类别、客户和通话信息。主管或处理人可分配工单，并按处理进展更新状态。

操作步骤：

1. 在通话中或通话结束后选择创建工单。
2. 填写工单标题、问题描述、优先级和类别。
3. 提交后查看系统生成的工单编号。
4. 根据处理情况将工单分配给处理人，并更新为待处理、处理中、已解决或已关闭。

结果反馈：工单创建后进入工单列表，并保留创建记录；状态更新后，系统保存处理历史；被分配人员会收到相应通知，便于继续跟进。

### 9. IVR 语音导航

IVR 语音导航用于在客户来电时播放欢迎语和按键提示，把客户引导到通用客服、技术支持、投诉处理等服务队列。客户没有按键或多次超时时，系统可按默认规则转人工服务。

操作步骤：

1. 管理员进入 IVR 管理或相关配置页面。
2. 查看默认流程，包括欢迎语、按键选择、转人工和超时处理。
3. 按业务需要设置按键对应的技能组，例如 1 转通用客服、2 转技术支持、3 转投诉处理。
4. 保存配置后，在测试来电中检查客户按键后的转接结果。

结果反馈：客户来电后会听到对应提示音，按键后进入匹配队列；未按键或超时后，系统按预设规则转人工或播放提示。

### 10. 录音、满意度与质检

录音、满意度与质检用于服务复盘。通话接通后，系统可记录录音信息；通话结束后可保存满意度评分；质检人员可根据通话记录和录音进行评分，并填写改进建议。

操作步骤：

1. 在通话记录中找到需要查看的通话。
2. 查看录音是否已生成，打开录音播放或录音信息页面。
3. 根据服务情况记录客户满意度评分。
4. 质检人员选择质检模板，填写评分项、备注和建议后提交。

结果反馈：录音信息会与通话记录关联；满意度保存后可用于统计；质检结果保存后，主管可查看不合格记录和改进建议。

### 11. 实时监控与报表大屏

实时监控页面面向主管和管理员。页面展示在线座席、通话中、空闲座席、队列等待、今日呼入、今日呼出、接通率、平均等待时长、座席状态表、呼叫趋势和座席状态分布。主管可以通过这些信息判断是否需要调配座席。

操作步骤：

1. 管理员或主管登录管理后台。
2. 默认进入实时监控页面，查看顶部关键指标。
3. 查看座席实时状态表，关注状态持续时间和今日接听量。
4. 查看呼叫趋势和状态分布图，需要展示到大屏时点击全屏按钮。

结果反馈：页面会定时刷新数据；队列等待超过阈值时显示预警标识；全屏模式适合投放到客服中心监控屏幕。

### 12. 消息推送与系统集成

消息推送用于把呼叫事件、座席状态变化、工单分配和提醒信息及时送到对应页面。系统集成用于让外部客户系统、企业管理系统或内部服务通过统一的数据通道读取或写入客户、座席、呼叫、工单和报表相关信息。

操作步骤：

1. 座席或主管保持页面在线，等待系统消息。
2. 客户来电、座席状态变化或工单分配时，页面接收并显示提醒。
3. 管理员为外部系统准备访问地址、账号或密钥。
4. 外部系统按约定调用客户、呼叫、工单或报表相关能力，并查看返回结果。

结果反馈：座席页面收到来电、工单或整理超时提醒；监控页面收到座席状态变化；外部系统请求成功后获得统一格式的业务数据或处理结果。

### 13. 部署、运维与监控告警

部署和运维功能用于让系统在开发、演示或生产环境中持续运行。项目提供一键 Docker 部署、Kubernetes 部署、服务日志、数据库维护、监控面板和告警规则等资料。运维人员可根据这些资料启动服务、查看运行状态、排查故障并进行备份恢复。

操作步骤：

1. 按部署说明准备服务器、配置文件和运行依赖。
2. 使用部署脚本或容器编排文件启动座席工作台、管理后台和系统服务。
3. 通过日志、健康检查、监控面板和告警规则查看服务状态。
4. 出现异常时，按运维手册检查服务、数据库、缓存、消息服务、电话服务和网关。

结果反馈：服务启动后，用户可访问座席工作台和管理后台；监控面板显示服务指标；出现异常时，告警和日志帮助运维人员定位问题。

## 五、典型使用流程

1. 管理员或主管登录管理后台，确认在线座席、空闲座席、队列等待和今日呼叫统计。
2. 座席打开座席工作台并登录，系统显示当前座席资料和空闲状态。
3. 客户来电后，系统按技能组、座席状态、客户等级和空闲时间进行分配。
4. 座席工作台弹出来电窗口，座席点击接听后进入通话中状态。
5. 通话过程中，座席可根据需要执行保持、恢复、静音、转接或三方通话。
6. 通话结束后，座席挂断电话，系统保存通话记录，座席可填写小结或创建工单。
7. 工单创建后，处理人按业务进展更新状态，主管可查看处理情况。
8. 主管通过实时监控页面查看队列等待、接通率和座席状态，必要时安排人员支援。
9. 质检人员基于通话记录、录音和满意度进行检查；运维人员按部署和监控资料维护系统运行。

## 六、使用注意事项

1. 座席接打电话前，应确认浏览器、网络和耳机设备正常。
2. 生产环境应修改默认账号密码和安全密钥，并限制不必要的访问入口。
3. 通话过程中不要重复点击拨打、挂断、转接等按钮，避免造成重复请求。
4. 创建客户资料和工单时，应尽量填写完整信息，便于后续查询和处理。
5. 主管应关注队列等待、空闲座席和接通率，避免客户长时间等待。
6. 运维人员应定期检查数据库、缓存、消息服务、电话服务、日志和备份情况。
