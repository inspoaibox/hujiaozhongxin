**适用对象：** 第一次使用呼叫中心系统的管理员、主管、座席、质检员、运维和测试人员  
**适用版本：** 1.0.0  
**最后更新：** 2026-05-11

## 阅读前说明

这份手册按“完全没用过系统的人也能照着点”的方式编写。每个页面都会说明入口、用途、页面内容、按钮含义、操作步骤和常见问题。

为避免误解，手册中会使用两个状态标记：

| 标记 | 含义 |
|------|------|
| 当前可用 | 当前前端页面或后端接口已经具备基础能力 |
| 标准完整产品 | 完整呼叫中心产品应具备的页面和功能，部分前端页面后续需要继续扩展 |

当前前端已经有这些页面：

| 系统 | 页面 | 地址 |
|------|------|------|
| 座席工作台 | 登录页 | `/login` |
| 座席工作台 | 呼叫处理页 | `/` |
| 管理后台 | 登录页 | `/login` |
| 管理后台 | 实时监控页 | `/` |

后端已经补齐基础接口的模块包括：认证、呼叫、客户、座席、工单、IVR、录音、质检、通知、报表。

## 1. 系统是什么

千牛云呼叫中心是一套网页系统，用来完成客服接电话、打电话、记录客户问题、生成工单、查看录音、做质检、看团队数据等工作。

可以把它理解为三部分：

| 部分 | 谁使用 | 做什么 |
|------|--------|--------|
| 座席工作台 | 客服座席 | 接电话、拨电话、转接、保持、挂断、查看提醒 |
| 管理后台 | 管理员、主管 | 看实时监控、管理人员、客户、工单、报表 |
| 后端服务 | 系统自动使用 | 认证、分配呼叫、保存数据、推送消息 |

## 2. 常用名词解释

| 名词 | 大白话解释 |
|------|------------|
| 座席 | 客服人员，也就是接打电话的人 |
| 主管 | 管理一组客服的人，负责看状态、处理预警和绩效 |
| 管理员 | 系统最高权限人员，负责账号、权限、配置 |
| 呼入 | 客户主动打进来的电话 |
| 呼出 | 座席主动打给客户的电话 |
| 软电话 | 页面里的电话面板，不是实体电话机 |
| 来电弹屏 | 客户来电时，页面弹出来电窗口 |
| 技能组 | 把客服按能力分组，比如通用客服、技术支持、投诉处理 |
| 队列 | 客户等待接通的排队区 |
| 保持 | 让客户暂时等待，座席去查资料 |
| 转接 | 把当前电话转给另一个座席 |
| 三方通话 | 座席、客户、第三个人一起通话 |
| 工单 | 记录客户问题和处理进度的任务单 |
| IVR | 电话里的语音菜单，例如“咨询请按 1” |
| 录音 | 系统保存的通话音频 |
| 质检 | 回听录音并给服务质量打分 |
| JWT Token | 登录成功后系统发给浏览器的身份凭证 |

## 3. 通用操作规则

### 3.1 登录规则

1. 打开系统地址。
2. 如果没有登录，会自动跳到登录页。
3. 输入账号和密码。
4. 点击“登录”。
5. 登录成功后进入系统首页。

默认管理员：

| 账号 | 密码 | 用途 |
|------|------|------|
| `admin` | `Admin@2025` | 初始化系统和演示登录 |

上线前必须修改默认密码。

### 3.2 常见按钮含义

| 按钮 | 作用 | 小白提示 |
|------|------|----------|
| 登录 | 提交账号密码 | 点一次即可，不要连续多次点击 |
| 退出 | 清除登录状态并回到登录页 | 下班或换人使用电脑前要退出 |
| 搜索/查询 | 按条件查数据 | 输入条件后点它 |
| 重置 | 清空筛选条件 | 查不到数据时可以先重置 |
| 新增/新建 | 创建一条新数据 | 例如新建客户、新建工单 |
| 编辑 | 修改已有数据 | 修改后要点保存 |
| 保存 | 保存当前填写内容 | 保存前检查必填项 |
| 取消 | 关闭弹窗或放弃修改 | 不会保存刚才输入的内容 |
| 删除 | 删除数据 | 点之前确认数据是否还能恢复 |
| 启用 | 让账号或配置生效 | 启用后可正常使用 |
| 禁用 | 暂停账号或配置 | 禁用后相关人员可能无法登录 |
| 导出 | 下载文件 | 常见为 Excel 或 PDF |
| 上传 | 从电脑选择文件上传 | 例如上传语音文件 |
| 播放 | 播放录音或语音 | 播放前确认电脑音量 |
| 发布 | 让配置正式生效 | 发布前要确认无误 |

### 3.3 表格怎么看

系统里很多页面都是表格。常见区域如下：

| 区域 | 说明 |
|------|------|
| 筛选区 | 页面上方输入条件的位置 |
| 列表区 | 中间显示数据行 |
| 操作列 | 每行右侧的查看、编辑、删除等按钮 |
| 分页区 | 页面底部切换页码和每页数量 |

### 3.4 弹窗怎么用

弹窗通常用于新增、编辑、确认操作。

| 元素 | 说明 |
|------|------|
| 标题 | 告诉你当前弹窗做什么 |
| 输入框 | 填写内容 |
| 下拉框 | 从已有选项里选择 |
| 确认/保存 | 提交操作 |
| 取消 | 关闭弹窗，不保存 |

## 4. 座席工作台登录页

**状态：当前可用**  
**页面地址：** 座席工作台 `/login`  
**使用人：** 座席、主管、测试人员

### 4.1 页面用途

用于登录座席工作台。登录成功后，系统会保存 Token、用户名、姓名、角色和座席 ID。

### 4.2 页面内容

| 区域 | 内容 | 说明 |
|------|------|------|
| 标题 | 座席登录 | 告诉用户当前是座席端登录 |
| 账号输入框 | 账号 | 输入用户名，例如 `admin` |
| 密码输入框 | 密码 | 输入登录密码 |
| 显示密码图标 | 眼睛图标 | 查看或隐藏密码 |
| 登录按钮 | 登录 | 提交账号密码 |
| 提示文字 | 默认管理员说明 | 提醒初始账号和上线前改密 |

### 4.3 登录步骤

1. 打开座席工作台地址，例如 `http://localhost:5173`。
2. 页面跳转到“座席登录”。
3. 在“账号”输入框输入账号。
4. 在“密码”输入框输入密码。
5. 如不确定密码是否输入正确，可以点密码框右侧的显示密码图标。
6. 点击“登录”。
7. 登录成功后进入“呼叫处理”页面。

### 4.4 按钮说明

| 按钮 | 点击后发生什么 | 失败时怎么办 |
|------|----------------|--------------|
| 登录 | 调用 `/api/auth/login`，成功后进入工作台 | 检查账号、密码、后端服务和 API 网关 |
| 显示密码 | 临时显示密码明文 | 旁边有人时不要使用 |

### 4.5 常见问题

| 问题 | 原因 | 处理 |
|------|------|------|
| 提示用户名或密码错误 | 输入不正确 | 重新输入，注意大小写 |
| 点登录没反应 | 后端或网关不可用 | 联系技术人员检查 auth-service 和 api-gateway |
| 登录后又回到登录页 | Token 没保存或被清除 | 重新登录，检查浏览器是否禁用本地存储 |

## 5. 座席工作台呼叫处理页

**状态：当前可用**  
**页面地址：** 座席工作台 `/`  
**使用人：** 座席、主管

### 5.1 页面用途

座席每天主要使用这个页面处理电话。它包含软电话、来电弹屏、通话控制、当前提醒和空闲座席列表。

### 5.2 页面总览

| 区域 | 内容 | 用途 |
|------|------|------|
| 顶部标题区 | 座席工作台、呼叫处理 | 告诉你当前页面 |
| 当前座席状态 | 姓名 + 状态标签 | 查看自己是否空闲、通话中等 |
| 退出按钮 | 退出 | 下班或换人时退出 |
| 软电话面板 | 拨号、接听、通话控制 | 处理电话 |
| 当前提醒 | 提醒列表 | 显示工单、整理超时等提醒 |
| 空闲座席 | 工号、姓名 | 转接时选择可用座席 |

### 5.3 顶部状态区

| 元素 | 说明 |
|------|------|
| 座席工作台 | 页面所属系统 |
| 呼叫处理 | 当前页面名称 |
| 姓名 · 状态 | 当前登录人员和状态 |
| 退出 | 清除 Token 并回到登录页 |

状态含义：

| 状态 | 含义 |
|------|------|
| 空闲 | 可以接电话 |
| 呼叫中 | 正在拨出，还没接通 |
| 通话中 | 已接通电话 |
| 保持中 | 客户正在等待 |
| 三方通话 | 已进入会议通话 |
| 整理 | 通话后处理记录 |
| 休息 | 暂时不接电话 |
| 离线 | 不在线 |

### 5.4 退出按钮

操作步骤：

1. 确认没有正在通话。
2. 点击右上角“退出”。
3. 系统断开 WebSocket。
4. 清除浏览器里的 Token。
5. 回到登录页。

注意：如果正在通话，不建议直接退出，应先挂断或完成转接。

## 6. 软电话：拨号区

**状态：当前可用**

### 6.1 拨号区什么时候出现

当没有正在处理的电话时，页面显示拨号盘。

### 6.2 拨号区内容

| 元素 | 说明 |
|------|------|
| 电话号码输入框 | 输入要拨打的客户号码 |
| 清空图标 | 清除输入框内容 |
| 拨打按钮 | 发起外呼 |
| 数字键盘 | 点击数字自动填入号码 |
| `*` 键 | 输入星号 |
| `#` 键 | 输入井号 |

### 6.3 手动输入号码拨打

1. 点击“输入电话号码”输入框。
2. 输入客户号码，例如 `13800138000`。
3. 点击“拨打”。
4. 系统创建外呼记录。
5. 页面进入“呼叫中”或“通话中”。

### 6.4 用数字键盘拨打

1. 依次点击数字按钮。
2. 输入框会自动拼接号码。
3. 输错时点击输入框里的清空图标重新输入。
4. 点击“拨打”。

### 6.5 拨打按钮

| 状态 | 点击结果 |
|------|----------|
| 未输入号码 | 提示“请输入电话号码” |
| 后端可用 | 调用 `/api/calls/outbound` 创建外呼 |
| 后端不可用 | 使用本地演示数据，避免页面空白 |
| 后端返回业务错误 | 页面显示错误原因 |

真实拨通电话需要接入 FreeSWITCH 和电话线路；未接入线路时只能演示页面流程。

## 7. 软电话：来电弹屏

**状态：当前可用**

### 7.1 什么时候出现

当 WebSocket 收到来电事件，并且事件类型为 `CALL_EVENT`、`eventType` 为 `INCOMING_CALL` 时出现。

### 7.2 弹屏内容

| 元素 | 说明 |
|------|------|
| 标题“来电” | 当前有新来电 |
| 头像图标 | 来电客户图标 |
| 客户姓名 | 有客户资料时显示姓名，没有则显示未知来电 |
| 来电号码 | 显示客户号码 |
| VIP 客户标签 | VIP 客户会显示黄色标签 |
| 接听按钮 | 接起电话 |
| 拒接按钮 | 拒绝本次来电 |

### 7.3 接听来电

1. 看到来电弹窗。
2. 查看客户姓名、号码和 VIP 标签。
3. 点击绿色“接听”。
4. 系统调用接听接口。
5. 弹窗关闭。
6. 页面进入通话控制面板。
7. 通话计时开始。

### 7.4 拒接来电

1. 看到来电弹窗。
2. 点击红色“拒接”。
3. 弹窗关闭。
4. 当前来电从页面清除。

注意：当前基础版拒接只清除前端来电状态；生产版应进一步通知呼叫服务把电话退回队列或挂断。

### 7.5 来电不弹窗怎么办

| 检查项 | 说明 |
|--------|------|
| ws-gateway | WebSocket 网关是否启动 |
| Kafka | 是否有呼叫分配事件 |
| agentId | 事件中的座席 ID 是否等于当前登录用户 ID |
| 前端地址 | `VITE_WS_URL` 是否配置正确 |
| 浏览器控制台 | 是否有 WebSocket 连接失败信息 |

## 8. 软电话：通话控制区

**状态：当前可用**

当电话进入呼叫中或通话中，拨号盘会变成通话控制区。

### 8.1 客户信息卡片

| 元素 | 说明 |
|------|------|
| 客户名称 | 有客户姓名时显示姓名，外呼无姓名时显示“外呼客户” |
| 客户号码 | 呼入显示来电号码，外呼显示被叫号码 |
| VIP 标签 | VIP 客户显示 |

### 8.2 通话状态栏

| 元素 | 说明 |
|------|------|
| 状态标签 | 空闲、来电、呼叫中、通话中、保持中等 |
| 通话计时 | 显示本次通话持续时间 |

### 8.3 保持按钮

图标：暂停/播放样式  
位置：通话控制按钮区  
可用条件：电话已接通或正在保持

作用：

| 当前状态 | 点击后 |
|----------|--------|
| 通话中 | 电话进入保持 |
| 保持中 | 恢复正常通话 |

操作步骤：

1. 通话中点击“保持”。
2. 按钮变为高亮。
3. 客户进入等待状态。
4. 查询资料或处理问题。
5. 再次点击按钮恢复通话。

注意：不要让客户保持太久，建议先告知客户“请稍等，我帮您查询一下”。

### 8.4 静音按钮

图标：麦克风  
可用条件：电话已接通

作用：

| 状态 | 说明 |
|------|------|
| 未静音 | 客户可以听到座席声音 |
| 已静音 | 客户听不到座席声音 |

当前基础版说明：静音按钮会切换页面状态；真实麦克风控制需要生产环境接入 SIP/WebRTC。

### 8.5 转接按钮

图标：分享/转接  
可用条件：电话已接通

点击后打开“转接呼叫”弹窗。

弹窗内容：

| 元素 | 说明 |
|------|------|
| 选择目标座席 | 下拉选择空闲座席 |
| 取消 | 关闭弹窗 |
| 确认转接 | 把当前电话转给目标座席 |

操作步骤：

1. 通话中点击“转接”。
2. 在下拉框选择目标座席。
3. 点击“确认转接”。
4. 转接成功后当前通话结束。
5. 页面回到空闲状态。

常见问题：

| 问题 | 处理 |
|------|------|
| 下拉框没有座席 | 当前没有空闲座席 |
| 转接失败 | 检查目标座席 ID、呼叫状态、后端 call-service |
| 按钮是灰色 | 当前电话还没接通 |

### 8.6 三方通话按钮

图标：连接/会议  
可用条件：电话已接通

点击后打开“发起三方通话”弹窗。

弹窗内容：

| 元素 | 说明 |
|------|------|
| 第三方电话号码输入框 | 输入要邀请的人 |
| 取消 | 关闭弹窗 |
| 发起三方 | 创建三方通话 |

操作步骤：

1. 通话中点击“三方通话”。
2. 输入第三方电话号码。
3. 点击“发起三方”。
4. 成功后通话状态变为三方通话。

注意：真实三方通话依赖电话网关能力。

### 8.7 挂断按钮

图标：红色电话  
作用：结束当前通话

操作步骤：

1. 确认客户问题已处理完。
2. 点击红色“挂断”。
3. 系统调用挂断接口。
4. 通话计时归零。
5. 页面回到空闲状态。

建议话术：“感谢您的来电，如有其他问题欢迎再次联系，再见。”

## 11. 管理后台登录页

**状态：当前可用**  
**页面地址：** 管理后台 `/login`  
**使用人：** 管理员、主管

### 11.1 页面用途

用于登录管理后台。登录成功后进入实时监控页。

### 11.2 页面内容和按钮

## 26. 日常工作流程

### 26.1 座席上班流程

1. 打开座席工作台。
2. 登录账号。
3. 检查耳机和麦克风。
4. 确认状态为空闲。
5. 等待来电或主动外呼。
6. 通话中按需要保持、转接、三方。
7. 通话结束后填写小结或创建工单。
8. 临时离开时切换休息。
9. 下班前确认没有通话。
10. 点击退出。

### 26.2 主管上班流程

1. 打开管理后台。
2. 登录账号。
3. 查看在线座席。
4. 查看队列等待。
5. 查看通话中和空闲比例。
6. 关注预警。
7. 分配或跟进工单。
8. 查看报表。
9. 下班前汇总数据。

### 26.3 管理员初始化流程

1. 登录默认管理员账号。
2. 修改默认密码。
3. 创建正式管理员账号。
4. 创建主管和座席账号。
5. 创建技能组。
6. 给座席分配技能组。
7. 配置 IVR。
8. 配置电话网关。
9. 配置录音存储。
10. 做一轮真实呼入、呼出、转接、录音测试。

## 27. 常见异常和处理

| 现象 | 可能原因 | 处理方法 |
|------|----------|----------|
| 页面空白 | 前端构建失败或资源路径错误 | 刷新页面，检查前端构建日志 |
| 登录失败 | 账号密码错误或 auth-service 不可用 | 检查账号、密码、认证服务 |
| 接口 401 | Token 缺失或过期 | 重新登录 |
| 点击拨打但没真实电话 | 未接入 FreeSWITCH 或线路 | 检查电话网关和线路 |
| 来电不弹屏 | WebSocket 或 Kafka 事件异常 | 检查 ws-gateway、Kafka、agentId |
| 转接失败 | 目标座席不可用或呼叫未接通 | 选择空闲座席，确认通话已接通 |
| 录音不能播放 | 录音文件不存在或 MinIO 不通 | 检查录音服务和对象存储 |
| 报表是演示数据 | report-service 不可用或聚合未接入 | 检查报表服务和数据源 |
| 工单通知收不到 | 通知服务或 WebSocket 推送异常 | 检查 notification-service 和 ws-gateway |

## 29. 给小白用户的安全提醒

1. 不要把账号密码告诉别人。
2. 第一次登录后尽快修改密码。
3. 离开座位前先退出或切换休息。
4. 通话中不要随意刷新页面。
5. 不确定按钮作用时，先问主管或管理员。
6. 生产环境不要使用默认密码和默认密钥。
7. 客户资料和录音属于敏感数据，不要随意下载或外传。
